# ENSIP-27 Explainer

This is a standalone Vocs site built with the same stack as [docs.ens.domains](https://docs.ens.domains): [Vocs](https://vocs.dev) for the docs framework, and design tokens sourced from [Thorin](https://thorin.ens.domains), ENS's design system.

It exists to make one draft proposal easy to read:

## [ENSIP-27: Node Classification and Metadata &rarr;](/ensip/27)

A standard for attaching structured, machine-readable metadata to ENS names and subnames &mdash; so a DAO, company, or individual can declare what role each subname plays (treasury, delegate, person, contract&hellip;) and what data it carries.

Currently in draft status, under review at [ensdomains/ensips#64](https://github.com/ensdomains/ensips/pull/64).
